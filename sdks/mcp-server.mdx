## Choose your deployment mode

You can connect to Cyberwave MCP in two ways:

1. **Hosted** (`https://mcp.cyberwave.com/mcp`)
   - No infrastructure to manage
   - Best for most users
2. **Self-hosted** (run your own `cyberwave-mcp-server`)
   - Full control over auth/networking/policies
   - Recommended for strict enterprise requirements

## Hosted MCP (`mcp.cyberwave.com`)

- **Endpoint:** `https://mcp.cyberwave.com/mcp`
- **Transport:** `streamable-http`
- **Protocol:** JSON-RPC 2.0 over MCP

### Hosted authentication (required for user-scoped access)

The official hosted deployment (`mcp.cyberwave.com`) expects a request token.
Use your API key from `https://cyberwave.com/profile` and send one of:

- `Authorization: Bearer <CYBERWAVE_API_KEY>` (recommended)
- `X-Cyberwave-Api-Key: <CYBERWAVE_API_KEY>`

### Hosted quick setup

You can grab your token at [https://cyberwave.com/profile](Your profile page)

#### VS Code

Add to your VS Code `settings.json` file:

```json
{
  "mcp": {
    "servers": {
      "cyberwave": {
        "url": "https://mcp.cyberwave.com/mcp",
        "headers": {
          "Authorization": "Bearer ${env:CYBERWAVE_API_KEY}"
        }
      }
    }
  }
}
```

Replace `${env:CYBERWAVE_API_KEY}` with your API key, or set the `CYBERWAVE_API_KEY` environment variable and restart VS Code.

#### Cursor

Edit your Cursor `mcp.json` configuration file to add the Cyberwave MCP server:

```json
{
  "mcpServers": {
    "cyberwave": {
      "url": "https://mcp.cyberwave.com/mcp",
      "headers": {
        "Authorization": "Bearer <CYBERWAVE_API_KEY>"
      }
    }
  }
}
```

Replace `<CYBERWAVE_API_KEY>` with your API key from your [profile page](https://cyberwave.com/profile).

#### Gemini CLI

```bash
gemini mcp add -t http cyberwave https://mcp.cyberwave.com/mcp \
  --header "Authorization: Bearer <CYBERWAVE_API_KEY>"
```

Replace `<CYBERWAVE_API_KEY>` with your API key from your [profile page](https://cyberwave.com/profile).

#### Claude Code

```bash
claude mcp add cyberwave -t http https://mcp.cyberwave.com/mcp \
  -H "Authorization: Bearer <CYBERWAVE_API_KEY>"
```

Replace `<CYBERWAVE_API_KEY>` with your API key from your [profile page](https://cyberwave.com/profile).

#### Codex CLI

Edit your `~/.codex/config.toml` and include the following:

```toml
[mcp_servers.cyberwave]
command = "npx"
args = ["-y", "mcp-remote@latest", "https://mcp.cyberwave.com/mcp", "--header", "Authorization: Bearer <CYBERWAVE_API_KEY>"]
```

Replace `<CYBERWAVE_API_KEY>` with your API key from your [profile page](https://cyberwave.com/profile).

#### Claude API (remote MCP connector)

```python
import anthropic

client = anthropic.Anthropic()

response = client.beta.messages.create(
    model="claude-opus-4-1",
    max_tokens=1024,
    messages=[{"role": "user", "content": "List available Cyberwave tools."}],
    mcp_servers=[
        {
            "type": "url",
            "name": "cyberwave",
            "url": "https://mcp.cyberwave.com/mcp",
            "authorization_token": "YOUR_CYBERWAVE_API_KEY"
        }
    ],
    tools=[{"type": "mcp_toolset", "mcp_server_name": "cyberwave"}],
)
```

#### OpenAI Agents SDK (Python)

```python
from agents import Agent, Runner
from agents.mcp import MCPServerStreamableHttp

mcp_server = MCPServerStreamableHttp(
    params={
        "url": "https://mcp.cyberwave.com/mcp",
        "headers": {"Authorization": "Bearer YOUR_CYBERWAVE_API_KEY"},
    },
    name="cyberwave",
)

agent = Agent(
    name="Cyberwave Assistant",
    instructions="Use Cyberwave MCP tools to inspect before acting.",
    mcp_servers=[mcp_server],
)

result = Runner.run_sync(agent, "List my available workspaces.")
print(result.final_output)
```

## Self-hosted MCP

Run your own server with:

```bash
cd cyberwave-clis/cyberwave-mcp-server
pip install -e .
cyberwave-mcp-server
```

### Self-hosted authentication options

You can authenticate backend calls in three main ways:

1. **Server token via environment**
   - `CYBERWAVE_API_KEY=<token>` (preferred)
   - or `CYBERWAVE_TOKEN=<token>`
2. **Credentials file fallback** (same format used by `cyberwave-cli`)
   - `/etc/cyberwave/credentials.json`
   - `~/.cyberwave/credentials.json`
   - or explicit `CYBERWAVE_CREDENTIALS_FILE=/path/to/credentials.json`
3. **Per-request token passthrough (HTTP transport)**
   - Client sends `Authorization: Bearer <token>`
   - Server uses that token for that request

If you want to strictly require request tokens in self-hosted HTTP mode, set:

`CYBERWAVE_MCP_REQUIRE_REQUEST_API_KEY=true`

## Best practices

1. Use Streamable HTTP for hosted/serverless deployments.
2. For custom clients, send both `Accept: application/json` and `Accept: text/event-stream`.
3. Use retries with exponential backoff for transient `5xx` and network failures.
4. Set tool timeouts and surface structured MCP error codes to users.
5. Use inspect-first workflows (`list/get` first, then `execute=false`, then `execute=true`).
