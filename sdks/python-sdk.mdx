## What is the Cyberwave Python SDK?

The Cyberwave Python SDK provides a powerful, easy-to-use interface for controlling robots, running simulations, and managing digital twins programmatically. Perfect for automation, testing, and advanced robotics applications.

The SDK supports two distinct deployment models — make sure you pick the right one before you start:

| Model | Where your code runs | Typical use cases |
| --- | --- | --- |
| **Remote application** | Any machine (laptop, server, cloud) | Dashboards, AI pipelines, teleoperation UIs, automation scripts |
| **On-device application** | Same hardware as the robot | Camera streaming, local inference, hardware-triggered alerts |

## Installation

<EarlyAccess />

Install the Cyberwave Python SDK using pip:

```bash
pip install cyberwave
```

<Note>
  **Requirements:** Python 3.11 or higher · API token from your Profile page (Private Beta access required)
</Note>

## Authentication & Setup

Get your API token from the Cyberwave dashboard and configure authentication:

<Steps>
  <Step title="Generate Token">
    Go to Dashboard → Profile → API Tokens to generate your token
  </Step>
  <Step title="Set Environment Variable">
    ```bash
    export CYBERWAVE_API_TOKEN=your_token
    ```
  </Step>
  <Step title="Initialize Client">
    The SDK will automatically pick up the environment variable:
    ```python
    from cyberwave import Cyberwave
    cw = Cyberwave()
    ```
  </Step>
</Steps>

## Remote Application

Use this model when your code runs on any machine and communicates with robots over the Cyberwave cloud.

### Quick start

```python
from cyberwave import Cyberwave

cw = Cyberwave(mode="simulation")

# affect() sets the locomotion/control target for subsequent commands
cw.affect("simulation")   # or cw.affect("real-world") for the physical robot

# Connect to your digital twin
robot = cw.twin("the-robot-studio/so101")

# If no default environment is configured, this auto-creates a "Quickstart Environment"
# and places the twin there.

# Move the twin's position in the environment editor
robot.edit_position(x=1.0, y=0.0, z=0.5)
robot.edit_rotation(yaw=90)

# For locomotion twins, send navigation commands
rover = cw.twin("unitree/go2")
rover.move_forward(distance=1.0)
rover.turn_left(angle=1.57)

# Actuate a joint to 30 degrees
robot.joints.set("1", 30)
```

<Note>
  Use `cw.affect("simulation")` to target the digital twin in Cyberwave, or
  `cw.affect("real-world")` to drive the physical robot. Individual calls can
  still override this with an explicit `source_type` argument.
</Note>

<Note>
  Runtime defaults: live clients publish state as `edge`, simulation clients
  publish state as `sim`, and locomotion/control helpers target `tele` or
  `sim_tele`. Legacy `source_type="sim"` control calls are still accepted for
  compatibility.
</Note>

`move()` and `rotate()` are deprecated for locomotion control. Use `move_forward()`, `move_backward()`, `turn_left()`, and `turn_right()` instead.

### Move a twin to another environment

If you create a robot first and later decide to place it in a different environment:

```python
from cyberwave import Cyberwave

cw = Cyberwave()
robot = cw.twin("the-robot-studio/so101")

robot.add_to_environment("TARGET_ENVIRONMENT_UUID")
```

`add_to_environment()` will:

- create a deep copy of the twin in the target environment
- mark the original twin as deleted
- delete the source environment if it has no twins left

## On-device Application

Use this model when your code runs **on the same machine as the robot** and needs direct access to local hardware — for example, opening the physical camera and streaming its feed to the Cyberwave platform.

<Note>
  If you want to write a **driver** (a Dockerized service that bridges a hardware device's native API to Cyberwave as twin metadata), use the `/cyberwave-driver` Claude Code skill instead — it is purpose-built for that workflow.
</Note>

### Stream video from the local camera

Install the additional dependencies for camera streaming:

```bash
pip install cyberwave[camera]
```

Then stream real-time video from the robot's camera to the Cyberwave platform via WebRTC:

```python
from cyberwave import Cyberwave

cw    = Cyberwave()
robot = cw.twin("the-robot-studio/so101")

robot.start_streaming()  # blocks until Ctrl+C
```

`start_streaming()` handles everything — it starts the WebRTC stream and blocks until you press Ctrl+C, then cleans up automatically. Pass `fps` and `camera_id` to override defaults:

```python
robot.start_streaming(fps=15, camera_id=1)
```

#### Advanced: non-blocking streaming

If you need to run other code alongside the stream (e.g. a local inference loop), use `stream_video_background()` instead. It is async and returns immediately with the streamer:

```python
import asyncio
from cyberwave import Cyberwave

async def run():
    cw    = Cyberwave()
    robot = cw.twin("the-robot-studio/so101")

    await robot.stream_video_background(fps=30)
    try:
        while True:
            # do other work here while the stream runs
            await asyncio.sleep(1)
    finally:
        await robot.stop_streaming()
        cw.disconnect()

asyncio.run(run())
```
