## Overview

Cyberwave supports importing datasets from multiple sources:

- **HuggingFace Hub** — Import datasets directly by repository ID
- **Zip Upload** — Upload pre-packaged dataset archives

Detection of dataset format happens automatically on the server side.

## Supported Formats

### Robot Datasets

| Format | Description |
|--------|-------------|
| LeRobot v3 | Latest LeRobot format with `meta/info.json` and chunked data |
| LeRobot v2.1 | Legacy LeRobot format with splits |
| RLDS/TFDS | TensorFlow Datasets format (common in robotics) |
| HDF5 | Robomimic and ACT/ALOHA style datasets |
| Zarr | Diffusion Policy and UMI style datasets |

### Other Formats

The detector emits a typed `source_format` on every imported dataset.
Values mirror [FiftyOne's built-in dataset types](https://docs.voxel51.com/user_guide/dataset_types.html)
where they overlap:

| Source format | Description |
|---------------|-------------|
| `coco_detection` | COCO object detection JSON |
| `yolov4` / `yolov5` | YOLO format datasets |
| `voc_detection` | Pascal VOC |
| `kitti_detection` | KITTI |
| `image_classification_directory_tree` | ImageNet-style class folders |
| `image_directory` / `video_directory` / `media_directory` | Plain media folders |
| `image_segmentation_directory` | Per-pixel masks alongside images |
| `cvat_image` / `cvat_video` | CVAT exports |
| `openlabel_image` / `openlabel_video` | OpenLABEL exports |
| `bdd`, `csv`, `dicom`, `geojson`, `geotiff` | Specialty formats |
| `unknown` | Detector couldn't classify the layout |

Source format is distinct from a dataset's *trainable artifact format*
(parquet, OpenVLA, LeRobot v2.1/v3) — those live on `ProcessedDataset`.

## Importing from HuggingFace Hub

To import a dataset from HuggingFace Hub:

1. Navigate to your workspace
2. Click **Import Dataset**
3. Select **HuggingFace Hub** as the source
4. Enter the repository ID (e.g., `lerobot/pusht`)
5. Click **Import**

Imports from HuggingFace are **lazy by default**: Cyberwave queries the
HuggingFace API for tags, card data and the file list, and (for LeRobot
datasets) reads only the small `meta/*.json` files. The dataset card
appears immediately with episode counts, FPS, robot type and cameras —
without copying the multi-GB payload anywhere.

The actual frames are fetched on demand later (e.g. when a training run
or visualization needs them). The dataset's `metadata.import.materialized`
flag stays `false` until a follow-up materialization step runs.

<Info>
For private HuggingFace repositories, ensure your organization has configured the HuggingFace token in the deployment settings.
</Info>

## Uploading Zip Files

To upload a dataset as a zip file:

1. Navigate to your workspace
2. Click **Import Dataset**
3. Select **Upload Zip** as the source
4. Select your zip file (max 10 GB)
5. Click **Upload**

The system will automatically detect the format after upload completes.

## Monitoring Progress

After initiating an import, you can monitor progress through:

- The dataset list view (status indicator)
- The dataset detail page (`metadata.upload_progress`)
