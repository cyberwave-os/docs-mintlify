## Goals

This guide walks you through setting up your Cyberwave environment, adding digital twins, and connecting everything, whether you're building applications or linking physical hardware. By the end, you will have:

- Set up a [virtual environment](/get-started/key-concepts#environments) and added a [digital twin](/get-started/key-concepts#digital-twins) of your robot
- Understood how to build applications with Cyberwave using the [Python SDK](/sdks/python-sdk), [MCP server](/sdks/mcp-server), or [REST API](/api-reference/rest/DefaultApi)
- Installed the [Cyberwave edge stack](/get-started/architecture#cyberwave-edge) on your device and paired physical hardware

<EarlyAccess />

### Step 1: Create an Environment

An **[environment](/get-started/key-concepts#environments)** is a 3D virtual space that mirrors your real-world robot setup. It's where your digital twins live, sensors stream data, and controllers send commands, all in real time.

Follow these steps to create a new environment: 
1. Go to the Cyberwave dashboard and click on **New Environment**. 
2. Give your environment a **name** and **description**. 

![](/images/new-env.png)

---

### Step 2: Add Digital Twins
A [digital twin](/get-started/key-concepts#digital-twins) is a virtual replica of your physical robot, it mirrors the robot's structure, joints, sensors, and behavior in real time within your environment.

You can add a digital twin from the Cyberwave catalog or import your own URDF file: 
1. Inside your environment, click **Add from Catalog** in the left panel.
2. Browse the catalog, find your robot's digital twin, and add it to the environment.
3. Position the twin within your environment to match your physical setup.

![](/images/digital-twins-catalog.png)

### Step 3: Build Applications with Cyberwave

With your environment and digital twins set up, you can start building applications immediately, **no physical hardware required**. Your code can run anywhere (laptop, cloud server, CI pipeline) and communicates with Cyberwave Cloud over the API.

<CardGroup cols={3}>

  </Card>
  <Card title="MCP Server" icon="plug" href="/sdks/mcp-server">

  </Card>

**Quick example: create a twin and control it with the Python SDK:**

```python
from cyberwave import Cyberwave

cw = Cyberwave(api_key="your_api_key")

robot = cw.twin("the-robot-studio/so101")
robot.edit_position(x=1.0, y=0.0, z=0.5)
robot.joints.set("shoulder_joint", 45, degrees=True)

frame = robot.capture_frame("numpy")
```

<Info>
You don't need physical hardware to get started. The SDK works with digital twins in simulation, control joints, capture frames, run workflows, and build your full application logic before connecting a real robot.
</Info>

## Connect Physical Hardware

The following steps walk you through installing the Cyberwave edge stack and pairing your physical robot with the digital twin you created above. If you're only working in simulation or building application code, you can skip these steps for now.

---

### Step 1: Install the Cyberwave CLI
The Cyberwave CLI is the command-line tool used to authenticate, pair, and manage your physical hardware with the Cyberwave platform.

<Tabs>
  <Tab title="Edge Device (Raspberry Pi, Jetson, etc.)">
    First, SSH into your edge device:

    ```bash
    ssh <edge-device-username>@<edge-device-ip>
    ```

    Then install the CLI:

    ```bash
    # Install the CLI (one time setup)
    curl -fsSL https://cyberwave.com/install.sh | bash
    ```
  </Tab>
  <Tab title="Local Computer">
    Open your terminal and run the following commands:

    ```bash
     # Install the CLI (one time setup)
    curl -fsSL https://cyberwave.com/install.sh | bash
    ```
  </Tab>
</Tabs>

---

### Step 2: Install the Edge Core

The [Edge Core](/get-started/architecture#cyberwave-edge) is the orchestrator of your Cyberwave edge components. It runs on edge devices (Raspberry Pi, Jetson, any Linux machine) and acts as the bridge between your physical hardware and the Cyberwave cloud backend.

```bash
sudo cyberwave edge install
```

The CLI will prompt you to log in with your Cyberwave credentials and then install the `cyberwave-edge-core` on your device.

---

### Step 3: Pair Hardware with Digital Twin

The CLI will walk you through pairing your physical robot with its digital twin. Follow the prompts in your terminal:

1. Select the environment you created earlier.
2. Select the digital twin you added to that environment.
3. The appropriate driver will be automatically installed and configured for your robot.

<Check>
Your physical robot is now paired with its digital twin and syncing in real time.
</Check>
