## Goals

This guide takes you from zero to a working Physical AI application. You will:

1. Create a virtual environment and add a digital twin of your robot
2. Install the Python SDK and control the twin in simulation from your laptop
3. Connect real hardware so the same code drives a physical robot

<EarlyAccess />

---

## Part 1 — Develop in simulation

Start here. No hardware required.

### Step 1: Create an Environment

An **[environment](/get-started/key-concepts#environments)** is a 3D virtual space that mirrors your real-world robot setup. It's where your digital twins live, sensors stream data, and controllers send commands, all in real time.

1. Go to the Cyberwave dashboard and click **New Environment**.
2. Give your environment a **name** and **description**.

![](/images/new-env.png)

---

### Step 2: Add a Digital Twin

A [digital twin](/get-started/key-concepts#digital-twins) is a virtual replica of your physical robot — it mirrors the robot's structure, joints, sensors, and behavior in real time within your environment.

1. Inside your environment, click **Add from Catalog** in the left panel.
2. Browse the catalog, find your robot, and add it to the environment.
3. Position the twin to match your physical setup.

![](/images/digital-twins-catalog.png)

### Step 3: Install the Python SDK

On your development machine:

```bash
pip install cyberwave
```

Then generate an API token at **Dashboard → Profile → API Tokens** and export it:

```bash
export CYBERWAVE_API_TOKEN=your_token
```

---

### Step 4: Control your twin in simulation

Run this from your laptop — no physical robot needed. The SDK targets the digital twin in simulation by default.

```python
from cyberwave import Cyberwave

cw = Cyberwave(mode="simulation")
cw.affect("simulation")

robot = cw.twin("the-robot-studio/so101")

robot.edit_position(x=1.0, y=0.0, z=0.5)
robot.joints.set("1", 30)
```

Open the Cyberwave dashboard — you should see your twin moving in the environment as the script runs.

<Check>
  Your application is running against a live digital twin in simulation. Iterate freely here before touching real hardware.
</Check>

---

## Part 2 — Connect real hardware

Once your application works in simulation, connect it to the physical robot. This part runs on the machine that is physically connected to the robot (an edge device like a Raspberry Pi or Jetson, or your laptop if the robot is plugged in directly via USB).

### Step 5: Install the Cyberwave CLI on the robot's machine

<Tabs>
  <Tab title="Remote edge device (Raspberry Pi, Jetson, etc.)">
    SSH into your edge device:

    ```bash
    ssh user@<edge-device-ip>
    ```

    Then install the CLI:

    ```bash
    curl -fsSL https://cyberwave.com/install.sh | bash
    ```
  </Tab>
  <Tab title="Robot connected directly to this machine">
    If your robot is connected via USB or serial (e.g. an SO101 arm plugged into your laptop), install the CLI on that same machine:

    ```bash
    curl -fsSL https://cyberwave.com/install.sh | bash
    ```
  </Tab>
</Tabs>

---

### Step 6: Install the Edge Core

The [Edge Core](/get-started/architecture#cyberwave-edge) runs on the robot's machine and bridges your physical hardware to the Cyberwave cloud.

```bash
sudo cyberwave edge install
```

The CLI will prompt you to log in and install `cyberwave-edge-core`.

---

### Step 7: Pair hardware with the digital twin

Follow the prompts in your terminal:

1. Select the environment you created in Step 1.
2. Select the digital twin you added in Step 2.
3. The appropriate driver will be automatically installed and configured for your robot.

<Check>
  Your physical robot is now paired with its digital twin and syncing in real time.
</Check>

Switch your application to target real hardware with a single line change:

```python
cw.affect("real-world")  # was: cw.affect("simulation")
```
